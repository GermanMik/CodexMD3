# Material 3 Skill Usage Notes

Use `$material` when the work is explicitly about Material Design 3 or Material You.

## Good triggers

- "Build a Compose Material 3 settings screen."
- "Generate an MD3 theme from this seed color."
- "Adapt this app shell for expanded width using MD3 navigation patterns."
- "Audit this Flutter screen for MD3 compliance."

## Non-triggers

- Generic CSS cleanup with no MD3 target.
- Brand design work unrelated to Material Design 3.
- Platform-agnostic UI feedback that does not need MD3 rules.

## Preferred workflow

1. Choose the primary mode: component, theme, layout, scaffold, or audit.
2. Load only the references needed for that mode.
3. Use the optional MCP tools for deterministic facts and scorecards.
4. Keep the final recommendation MD3-specific and platform-aware.
